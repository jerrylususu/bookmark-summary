## TL;DR

## Summary
TOON（Token-Oriented Object Notation）是一种面向大语言模型（LLM）的紧凑、人类可读的数据序列化格式，旨在替代 JSON 以减少输入提示的令牌数量。它结合了 YAML 的缩进结构和 CSV 的表格布局，特别适用于处理结构统一的数组对象。

### 核心理念
- **令牌高效**：相比标准 JSON，TOON 可节省 30–60% 的令牌，从而降低 LLM 使用成本。
- **支持结构验证**：通过显式声明数组长度和字段名，帮助 LLM 解析和验证数据。
- **格式简化**：减少冗余标点（如大括号、引号），采用缩进和表格形式提升可读性。

### 适用场景
- 最佳用于**结构统一的对象数组**（如员工记录、时间序列数据）。
- 在嵌套或非均匀数据结构中，JSON 可能更高效；纯表格数据中，CSV 的令牌开销更小。
- 适用于对令牌成本敏感但需保持结构清晰的 LLM 应用。

### 基准测试表现
- **准确性与效率**：在多个数据集上，TOON 平均准确率达 73.9%（高于 JSON 的 69.7%），同时令牌数减少 39.6%。
- **模型兼容性**：在 GPT-5、Gemini 等主流模型上表现稳定，尤其在结构化查询（如字段检索、聚合）中优势明显。
- **局限性**：对深度嵌套或半结构化数据，节省效果有限；在延迟敏感场景需实测性能。

### 工具与使用
- **TypeScript SDK**：提供 `encode`/`decode` API，支持与 JSON 互转。
- **CLI 工具**：通过 npx 直接使用，支持文件或标准输入/输出转换。
- **在线体验**：社区提供 Playground 用于格式比较和验证。

TOON 目前处于生产就绪但持续演进阶段，鼓励通过贡献规范或反馈参与改进。
