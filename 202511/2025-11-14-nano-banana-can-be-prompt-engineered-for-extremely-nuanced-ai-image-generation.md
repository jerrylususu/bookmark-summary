# Nano Banana can be prompt engineered for extremely nuanced AI image generation

- Added At: 2025-11-14 13:13:34
- Tags: #read #llm #guide

## TL;DR
Google推出的Nano Banana图像生成模型基于自回归技术，能精确遵循复杂提示并生成高质量图像，如处理荒诞请求和多指令编辑。模型优势包括强大的文本理解和免费使用，但存在风格转换弱、渲染文字出错及生成速度慢等局限。

## Summary
本文介绍了Google推出的图像生成模型Nano Banana（正式名称为Gemini 2.5 Flash Image），重点探讨了其出色的提示遵循能力、技术原理、应用方法及局限性。

### 模型背景与特点
- **技术基础**：Nano Banana是一种自回归模型，与扩散模型不同，它通过生成令牌再解码为图像，类似于ChatGPT的文本生成方式。
- **优势**：凭借强大的文本编码器，Nano Banana在提示遵循方面表现卓越，能精准处理复杂、细粒度的指令。
- **可访问性**：用户可通过Gemini网页或移动应用免费生成图像，开发者可使用Gemini API（每张图像成本约0.04美元），作者还开源了Python包`gemimg`简化API调用。

### 提示遵循能力测试
通过多个测试案例展示了Nano Banana的强提示遵循能力：
- **荒诞提示测试**：如生成“骷髅形状的煎饼”，模型能准确满足形状、配料和逻辑细节（如糖浆滴落）。
- **多重编辑测试**：同时执行五个编辑指令（如添加草莓、更换盘子），仅修改必要部分，保持其他内容不变。
- **主题一致性测试**：输入多张“丑版索尼克”图片，模型能生成其与奥巴马握手的图像，并通过添加“普利策奖获奖照片”等提示提升构图质量。
- **复杂规则测试**：使用Markdown列表、JSON结构或详细描述（如指定三只小猫的毛发颜色、位置规则），模型能全面遵守约束。

### 技术原理分析
- **文本编码优势**：Nano Banana基于Gemini 2.5 Flash的多模态编码器，训练数据包含大量Markdown、JSON和图像标注，使其能理解超出典型图像标题的复杂提示。
- **上下文窗口**：支持32,768个令牌，远高于传统模型（如CLIP的77个令牌），便于多轮对话和复杂提示处理。
- **系统提示泄露测试**：通过对抗性提示发现模型可能内置了防止滥用关键词的规则，但文本生成能力主要源于基础编码器而非提示重写器。

### 高级应用示例
- **网页渲染**：输入HTML/CSS/JavaScript代码，模型能大致还原网页布局和样式。
- **JSON结构化描述**：使用详细JSON描述人物特征（如职业混合体），模型能生成符合要求的图像，但需通过提示工程（如指定摄影设备）避免数字插画风格。

### 局限性
- **风格迁移能力差**：在处理如“Studio Ghibli风格转换”提示时效果不佳，自回归特性可能限制了风格适应性。
- **文本渲染不完美**：生成代码或文字时可能出现拼写错误或格式问题。
- **生成速度慢**：高质量图像生成需约30秒，与ChatGPT图像生成类似。

总结：Nano Banana在提示遵循和复杂指令处理上领先，尤其适合需要高精度的应用，但风格转换和文本生成仍有改进空间。
