# How good engineers write bad code at big companies

- Added At: 2025-11-29 05:14:58
- Tags: #read #career #people

## TL;DR
大公司代码质量差主要源于组织结构问题：高人员流动率、忽略专业积累、资深工程师负担过重及工作压力，公司为了组织灵活性牺牲代码质量。个人难以改变，需积极适应和积累专长。

## Summary
在大公司中，优秀工程师写出糟糕代码的主要原因是组织结构和流程设置，而非个人能力问题。核心因素包括：

- **高人员流动率**：大公司员工平均任期仅为1-2年，工程师频繁更换团队或项目，导致多数代码修改由对代码库、语言或系统不熟悉的“初学者”完成。
- **缺乏长期专业积累**：公司刻意将工程师视为可替换资源，通过内部调动优先满足短期业务需求（如快速转向AI项目），牺牲了在特定系统上的深度专业知识积累。
- **资深工程师负担过重**：少数经验丰富的“老手”需兼顾自身工作和代码审查，但时间有限，无法全面把关，且公司未正式赋予其维护代码质量的职责。
- **工作环境压力**：工程师常在紧迫截止日期下处理陌生代码库，只能追求“足够好”的解决方案，而非最优设计。

公司对此有意识地进行权衡：用代码质量换取组织灵活性。这反映了“不纯工程”的现实——工程师像水电工一样，在陌生系统中应对突发问题，错误难免。个人工程师无力改变这一体系，唯一能做的是在特定领域积累专业知识，尽力引导合理技术决策。根本问题在于大公司迫使工程师在陌生环境中工作，而非其能力不足。
