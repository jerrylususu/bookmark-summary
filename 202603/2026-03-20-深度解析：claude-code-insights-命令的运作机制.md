## Summary
深度解析：Claude Code /insights 命令的运作机制

**命令与输出**
- 命令：`/insights`
- 描述：生成 Claude Code 会话分析报告
- 输出：交互式 HTML 报告，保存于 `~/.claude/usage-data/report.html`

**分析流程（六阶段）**
1. **收集与过滤**：抓取 `~/.claude/projects/` 下的会话日志，剔除智能体子会话、内部操作、短会话（消息少于2条或时长少于1分钟）。
2. **元数据提取**：解析每个会话的 Token 用量、工具使用、时长、编程语言、Git 活动、用户打断、错误等结构化数据。
3. **长对话摘要**：若会话记录超过 30,000 字符，切分为 25,000 字符片段并摘要，以提取关键信息。
4. **特征提取**：使用 Haiku 模型分析会话，提取定性特征维度（如目标类别、用户满意度、卡点类型、帮助程度等），结果缓存以加速后续运行。
5. **聚合与分析**：汇总所有会话数据，通过多个专用 Prompt 进行深度分析，包括项目领域、交互风格、亮点、卡点、改进建议、未来展望等。
6. **报告生成**：将数据和洞察渲染为交互式 HTML 报告，包含统计仪表盘、可视化图表和叙述部分。

**关键特征维度**
- **目标类别**：如调试、功能实现、Bug 修复、代码重构等。
- **用户满意度**：从受挫到开心的层级。
- **卡点类型**：如理解错误、方法错误、代码 Bug、用户拒绝等。
- **帮助程度**：从无帮助到不可或缺。
- **会话类型**：如单任务、多任务、迭代优化等。

**技术细节**
- 模型：Haiku
- 每个 Prompt 最大 Token：8192
- 每次运行分析会话数：最多 50 个新会话
- 记录大小限制：30,000 字符
- 摘要分块大小：25,000 字符

**隐私说明**
- 所有分析通过 Anthropic API 在本地进行，数据保留在本地机器。
- 特征提取关注交互模式，而非代码内容。

**获取更佳洞察的技巧**
1. 多用 Claude Code 以丰富会话数据。
2. 提供反馈（如“谢谢”或“不对”）以追踪满意度。
3. 保持自然使用习惯。
4. 定期运行以查看模式演变。
