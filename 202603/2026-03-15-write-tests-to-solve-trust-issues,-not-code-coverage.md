# Write tests to solve trust issues, not code coverage
- URL: https://dodov.dev/blog/write-tests-to-solve-trust-issues-not-code-coverage
- Added At: 2026-03-15 11:09:21
- Tags: #read

## TL;DR
本文强调编写测试旨在解决信任问题，而非追求代码覆盖率。作者主张测试应针对真实生产行为，基于实际需求决定，而非盲目追求数字；在组织中，信任应通过代码审查和团队协作分配，最终提升开发信心与效率。

## Summary
这篇文章的核心观点是：编写测试的目的是为了解决信任问题，而非追求代码覆盖率。作者通过个人经历说明，测试应针对真实生产环境中的具体行为，而非假设的边缘情况。他认为，测试是解决主观信任问题的手段，而非技术指标。高覆盖率并不等同于代码稳定，测试应基于实际需求和信任程度来决定，而非盲目追求数字。在组织层面，信任应通过代码审查和团队协作来分配，而非依赖覆盖率政策。最终，测试应服务于提升对代码的信心，从而提高开发效率和幸福感。
