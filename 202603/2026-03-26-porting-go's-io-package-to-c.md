# Porting Go's io package to C
- URL: https://antonz.org/porting-go-io/
- Added At: 2026-03-26 14:17:39
- Tags: #read #deepdive

## TL;DR
本文介绍了将Go的`io`包移植到C语言的过程，通过结构体模拟切片、联合体处理多返回值、函数指针实现接口等方法，在C中构建Go子集，使代码风格相似但更冗长，并计划后续移植更多包。

## Summary
本文介绍了将Go语言标准库中的`io`包移植到C语言的过程。作者首先说明了移植的动机：为了在C中使用Go风格的代码，并逐步构建一个Go子集。由于`io`包依赖较少，因此作为首个移植目标。

文章详细阐述了在C中模拟Go特性的设计决策：
1. **切片（Slices）**：Go的切片在C中通过结构体实现，包含指针、长度和容量，并添加了边界检查宏。
2. **多返回值**：Go的多返回值（如`(n int, err error)`）在C中通过联合体（`so_Value`）和结构体（`so_Result`）来模拟，将值和错误打包返回。
3. **错误处理**：采用简单的指针字符串表示错误，使用哨兵错误（如`io_EOF`），通过指针比较而非字符串内容来判断错误类型。
4. **接口（Interfaces）**：Go的接口在C中通过包含函数指针和`self`指针的“胖”结构体实现，方法调用通过函数指针完成。
5. **类型断言**：Go的动态类型断言在C中被简化，通过比较函数指针来判断类型，但放弃了完整的动态类型系统。
6. **具体类型实现**：以`LimitedReader`为例，展示了如何将Go结构体和方法移植到C，包括逻辑和接口调用的转换。
7. **`Copy`函数**：结合上述特性，实现了Go的`Copy`函数，使用栈分配缓冲区，并处理多返回值和错误。

最后，作者总结了移植过程：通过结构体、联合体、函数和宏解决了切片、多返回值、错误和接口等问题，使C代码结构与Go相似但更冗长。下一步计划移植`bytes`和`strings`包，并邀请读者尝试作者的项目Solod，该项目允许编写可翻译为C的Go代码。
