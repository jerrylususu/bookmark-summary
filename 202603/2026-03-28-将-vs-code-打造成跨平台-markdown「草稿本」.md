# 将 VS Code 打造成跨平台 Markdown「草稿本」
- URL: https://hsu.cy/2023/09/vscode-as-scratchpad/
- Added At: 2026-03-28 07:50:32
- Tags: #read #tips

## TL;DR
本文介绍如何利用 VS Code 的配置文件、代码片段和任务功能，将其打造成跨平台 Markdown 草稿本，实现快速记录、自动保存及多端同步。

## Summary
本文介绍了如何将 VS Code 打造成一个跨平台的 Markdown 草稿本，以满足快速记录、编辑和同步临时文本的需求。

**需求与方案选择**
作者认为传统笔记或草稿工具在跨平台同步、编辑效率和设置隔离方面存在不足。经过比较，选择 VS Code 作为基础，因其跨平台、开源、插件生态丰富，且支持配置文件（Profile）功能，能为草稿本创建独立的环境，不影响代码编辑等其他用途。

**搭建基本框架**
1.  **确定存储目录**：选择一个可同步的路径（如云盘），用于存放草稿文件。
2.  **创建专用配置文件**：在 VS Code 中通过命令面板创建新的配置文件（如命名为 "flashes"），建议从零开始或使用 "Doc Writer" 模板，以隔离草稿本设置。

**优化草稿本内部环境**
通过专用配置文件进行以下优化，不影响默认设置：
*   **精简界面布局**：隐藏无关面板，启用 Zen 模式，使界面更清爽。
*   **设置自动保存**：启用 `files.autoSave`（如每5秒）和 `files.hotExit`，避免手动保存和丢失未保存内容。
*   **一键插入格式模板**：利用 VS Code 的代码片段（Snippet）功能，为常用 Markdown 格式（如任务列表、表格）创建模板，并绑定快捷键（如 `Cmd/Ctrl+K T` 插入任务列表）。
*   **让新建文件名附带时间戳**：通过 VS Code 的任务（Task）功能，编写脚本实现新建文件时自动添加时间戳文件名，并将默认的 `Cmd/Ctrl+N` 快捷键绑定到该任务。

**快速启动和抓取文本**
*   **快速启动草稿本**：通过命令行 `code /路径 --profile 配置文件名` 创建快捷方式（如 macOS 的快捷指令或 Windows 的快捷方式），一键启动草稿本环境。
*   **快速捕获别处内容**：
    *   **macOS**：使用快捷指令（Shortcuts）制作服务，可将选中文本或剪贴板内容快速保存为 Markdown 草稿文件。
    *   **Windows**：通过 PowerShell 脚本捕获剪贴板内容，保存为带时间戳的 Markdown 文件，并用快捷方式调用。

通过以上步骤，VS Code 可以成为一个高效、跨平台的 Markdown 草稿本，兼顾快速记录、编辑和同步需求。
