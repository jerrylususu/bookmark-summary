# Comprehension Debt — the hidden cost of AI generated code.
- URL: https://addyosmani.com/blog/comprehension-debt/
- Added At: 2026-03-15 11:17:51
- Tags: #read #agent

## TL;DR
理解债指过度依赖AI导致人类理解与代码增长脱节，易引发系统性风险。被动使用AI会削弱开发者能力，主动提问可缓解。未来关键领域需强化对AI代码的理解，工作重心应从代码量转向深度理解与决策。

## Summary
理解债是指过度依赖AI和自动化导致的人类智能与记忆的隐性成本，具体表现为代码量增长与人类实际理解程度之间的差距扩大。与技术债不同，理解债不易被察觉，它滋生虚假信心，可能在关键时刻引发系统性问题。

AI生成代码的速度远超人类评估能力，打破了传统代码审查的教育与质量控制作用。测试和规范虽能缓解问题，但无法完全替代人类对系统深层逻辑的理解。研究表明，被动使用AI编码工具会显著降低开发者的理解能力，而主动提问式使用则能保持较高水平。

随着AI代码量增加，真正理解系统的工程师价值反而上升。当前组织的度量体系（如开发速度、PR数量）无法捕捉理解债，使其比技术债更危险。未来在医疗、金融等关键领域，监管将要求团队对AI生成代码有充分理解。

应对理解债需重构工作重点：从追求代码生成量转向确保对交付内容的深度理解。这包括明确变更意图、将验证作为结构性约束、维护系统级心智模型，并诚实区分“测试通过”与“真正理解”。理解工作是开发的核心，AI负责翻译，但人类必须负责理解与决策。
