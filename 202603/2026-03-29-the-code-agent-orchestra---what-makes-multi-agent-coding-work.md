## TL;DR
本文介绍了多智能体编码从单智能体同步协作转向异步协同的范式转变，提出了三种模式（子智能体、团队、大规模编排）及质量保障、工具生态和实践方法，旨在提升编码效率与质量。

## Summary
本文介绍了多智能体编码的协调模式与实践方法，核心是从单智能体同步协作转向多智能体异步协同的范式转变。作者提出了从“指挥者”到“协调者”的思维转变，并详细阐述了三种主要模式、工具生态及质量保障机制。

### 核心范式转变
- **从单智能体到多智能体**：开发者从与单个AI助手同步交互，转变为协调多个拥有独立上下文和职责的异步智能体团队。
- **从指挥者到协调者**：工作模式从实时指导单个智能体（指挥者），转变为规划、分配任务并定期检查的团队管理（协调者）。

### 多智能体模式
1. **子智能体模式**：最简单的多智能体模式。父智能体分解任务，生成并管理子智能体（如数据层、业务逻辑、API路由），手动管理依赖关系。优点是并行执行、上下文隔离，但缺乏自动协调。
2. **智能体团队模式**：在子智能体基础上增加协调原语。包含团队领导、共享任务列表（带依赖跟踪和文件锁定）和队友。队友通过共享列表认领任务，并通过点对点消息直接通信，避免领导成为瓶颈。推荐3-5人团队。
3. **大规模编排模式**：适用于管理5个以上智能体或跨仓库任务。工具分为三层：
   - **层1**：进程内子智能体和团队（如Claude Code）。
   - **层2**：本地编排器（如Conductor、Vibe Kanban），提供可视化仪表盘和差异审查。
   - **层3**：云端异步智能体（如Claude Code Web、GitHub Copilot Coding Agent、Jules、Codex Web），分配任务后关闭笔记本，返回时即可看到PR。

### 关键实践与质量保障
- **质量门控**：
  - **计划审批**：智能体编码前先提交计划，由领导审查，防止架构错误。
  - **钩子机制**：在任务完成等生命周期事件自动运行测试、代码检查，失败则继续工作。
  - **AGENTS.md**：记录模式、陷阱和风格偏好，每会话读取并更新，实现知识复合增长。
- **可靠性技巧**：
  - **循环防护**：设置最大迭代次数（如8次），强制每次重试前反思错误。
  - **专用评审智能体**：自动触发代码评审，确保领导只看到已验证的代码。
- **成本与效率优化**：
  - **多模型路由**：根据任务类型（规划、实现、评审）选择不同成本模型。
  - **工作树生命周期脚本**：自动化创建、合并和清理工作树。
  - **人类维护AGENTS.md**：研究显示，人类编写的AGENTS.md比AI生成的更有效，能提升成功率。

### 工具生态亮点
- **Conductor**：Mac端本地编排工具，支持多智能体并行运行与可视化审查。
- **Claude Code Web**：云端异步智能体，浏览器操作，适合委托后离开。
- **GitHub Copilot Coding Agent**：GitHub原生异步智能体，与Issues和PR深度集成。
- **Jules**：Google的云端智能体，支持计划审批和音频变更日志。
- **Codex Web**：提供可验证证据（终端日志和测试输出），便于审计。

### 核心原则与纪律
- **瓶颈转移**：生成速度不再是瓶颈，验证（如测试、代码审查）成为关键。
- **委托任务，而非判断**：让智能体处理有明确标准的任务（如样板代码、迁移），人类保留架构设计、系统理解和决策权。
- **规范即杠杆**：模糊需求会放大错误，精确规范能提升整个智能体舰队的输出质量。
- **工厂模型**：将软件开发视为工厂生产线，包括计划、生成、监控、验证、集成和回顾六个步骤。

### 五个立即实践的模式
1. **子智能体分解任务**：使用Task工具生成专注子智能体。
2. **智能体团队实现并行**：启用实验性团队功能，创建领导+3队友。
3. **Git工作树隔离**：每个智能体独立工作树，避免冲突。
4. **质量门控保障信任**：计划审批、钩子机制、人工审查。
5. **AGENTS.md复合学习**：记录并更新知识，每会话复用。

通过以上模式和实践，开发者可以高效协调多智能体团队，提升编码效率与质量，同时保持对系统的理解和控制。
