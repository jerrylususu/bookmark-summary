# AI=true is an Anti-Pattern
- URL: https://keleshev.com/ai-equals-true-is-an-anti-pattern
- Added At: 2026-03-06 14:36:06
- Tags: #read

## TL;DR
文章批评了编程中针对AI设计文档、工具和工作流的反模式，主张统一接口与通用设计，以兼顾人类与AI，提升协作效率和互操作性。

## Summary
文章指出，当前编程实践中出现了一种反模式：将文档、工具和工作流专门针对AI进行设计，例如将文档放在`copilot-instructions.md`或`AGENTS.md`中，或通过`AI=true`等标志区分人类与AI的工作流。作者认为这种做法是错误的，因为：

1. **文档应兼顾人类与AI**：良好的文档对人类和AI同样重要，应放置在双方都能轻松发现的地方，如`README.md`，而非仅针对AI的文件。
2. **工具应统一接口**：命令行工具和API对人类和AI都友好，应优先使用，而非专门开发MCP工具或GUI工具，以避免不必要的复杂性和上下文窗口浪费。
3. **避免隔离工作流**：不应通过参数（如`AI=true`）区分人类和AI的工作流，而应设计通用工具，减少上下文和认知负担。
4. **保持互操作性**：人类与AI在文本接口和惯例上有足够相似性，应尽量保持统一，以促进协作和自动化。

总之，作者呼吁编程实践应统一人类与AI的工作流，避免反模式，以提升整体效率和互操作性。
