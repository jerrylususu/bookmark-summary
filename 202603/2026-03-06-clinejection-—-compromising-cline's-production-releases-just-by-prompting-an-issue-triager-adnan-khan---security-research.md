## TL;DR
Cline 工具因 AI 代理提示注入漏洞被利用，攻击者通过缓存污染窃取发布凭证，导致恶意版本发布。影响数百万开发者，团队已修复并加强安全措施。

## Summary
**漏洞概述**  
Cline 是一款开源 AI 编码工具，集成于 VS Code 等 IDE。2025 年 12 月 21 日，Cline 在 GitHub Actions 中引入 AI 代理自动分类问题，但该代理存在提示注入漏洞。攻击者可通过创建问题触发提示注入，利用 GitHub Actions 缓存污染技术，窃取生产发布凭证（VSCE_PAT、OVSX_PAT、NPM_RELEASE_TOKEN），进而向数百万开发者发布恶意更新。

**攻击链**  
1. **提示注入**：问题标题直接嵌入 AI 提示，攻击者可构造恶意标题诱导 AI 执行任意命令（如安装恶意 npm 包）。  
2. **缓存污染**：利用 GitHub Actions 缓存策略（2025 年 11 月更新后支持快速驱逐），攻击者可填充缓存并替换合法条目，使后续工作流加载恶意缓存。  
3. **凭证窃取**：通过污染夜间发布工作流的缓存，获取生产发布凭证。由于夜间凭证与生产凭证绑定同一发布者，攻击者可发布恶意版本。  
4. **供应链攻击**：窃取凭证后，攻击者可向 VS Code Marketplace、OpenVSX 和 NPM 发布恶意更新，影响所有自动更新的用户。

**影响**  
- 若凭证被窃取，攻击者可向数百万开发者发布恶意扩展或 CLI 工具，窃取敏感数据（如 SSH 密钥、源代码）。  
- 2026 年 2 月 17 日，已发生未经授权的 NPM 发布（版本 2.3.0），添加了恶意 `postinstall` 脚本。

**时间线**  
- **2025 年 12 月 21 日**：漏洞引入。  
- **2026 年 1 月 1 日**：研究员通过 GitHub 私有漏洞报告和邮件联系 Cline，未获响应。  
- **2026 年 2 月 9 日**：公开披露后，Cline 在 1 小时内修复漏洞。  
- **2026 年 2 月 10 日**：Cline 旋转凭证并审计发布记录，确认无未授权版本。  
- **2026 年 2 月 17 日**：NPM 凭证未完全撤销，导致恶意版本发布，后紧急撤销并发布安全公告。

**缓解措施**  
- **用户**：禁用 Cline 自动更新，固定已知安全版本。  
- **Cline 团队**：  
  - 移除问题分类工作流的高权限工具（如 Bash、Write）。  
  - 避免在涉及生产凭证的工作流中使用缓存。  
  - 隔离夜间与生产发布环境（使用不同命名空间和凭证）。  
  - 建立漏洞响应流程，确保及时处理安全报告。

**结论**  
Clinejection 展示了 AI 代理漏洞如何成为 CI/CD 攻击的入口，结合提示注入、缓存污染和凭证窃取，形成完整的供应链攻击链。开发者工具公司需加强安全实践，及时响应漏洞报告，避免类似风险。
