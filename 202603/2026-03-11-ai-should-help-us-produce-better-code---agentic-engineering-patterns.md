# AI should help us produce better code - Agentic Engineering Patterns

## TL;DR
AI工具应提升而非降低代码质量，通过处理技术债务、辅助决策和复盘优化，实现高效可持续的开发。

## Summary
文章指出，AI工具应帮助开发者产出更优质的代码，而非导致代码质量下降。作者认为，若采用编码代理后代码质量降低，应直接优化开发流程，而非归咎于工具。技术债务常源于耗时但简单的修复任务，如API调整、命名清理或代码重构，而编码代理能高效处理这些任务，降低改进成本。

AI工具还能拓展开发者的选项，通过生成常见解决方案和辅助探索性原型设计，帮助做出更优的技术决策。例如，通过快速构建模拟系统进行负载测试，验证技术选型的可行性。

文章强调“复合工程循环”的重要性：在每次项目后进行复盘，提炼有效方法并文档化，以持续提升代码质量和代理效能。最终，AI使开发者能兼顾新功能交付与代码质量提升，实现高效且可持续的工程实践。
