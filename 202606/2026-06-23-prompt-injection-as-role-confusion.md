# Prompt Injection as Role Confusion

- Added At: 2026-06-23 14:37:59
- Tags: #read #llm

## TL;DR
该文指出，大语言模型依赖文本“风格”而非系统标签来感知角色，导致提示注入等安全漏洞。角色标签本用于隔离目标冲突，但现有设计存在缺陷，需通过系统性研究来增强其安全性。

## Summary
这篇文章将提示注入攻击重新解释为大语言模型（LLM）对“角色”的感知混淆。核心观点是：LLM并非通过系统性的角色标签，而是通过文本的表面“风格”来识别角色。这种不安全的感知方式是导致提示注入漏洞的根本原因，并催生了如“思维链伪造”等新型攻击。

**主要内容总结如下：**

1.  **问题根源：角色感知缺陷**：LLM接收的是一个包含所有内容（用户指令、工具输出、自身思考）的连续文本流，并依赖“角色标签”（如`system`, `user`, `tool`, `assistant`, `think`）来划分区域并定义每部分文本的权威性。然而，研究通过探测发现，模型主要依据文本的**写作风格**（如推理式的语言风格）而非真正的标签来判定角色。例如，即使移除或错误标记标签，类似推理的文本仍被模型内部视为自己的思考。

2.  **攻击新解与验证**：传统的提示注入（如在工具返回的网页中隐藏指令）之所以成功，正是因为模型将伪装成用户命令风格的工具文本误认为真实指令。更强大的攻击“思维链伪造”利用了这一点：通过在用户消息中伪造具有特定推理风格的文本，诱导模型将其当作自己已得出的结论并执行，从而绕过安全限制。实验表明，攻击成功率与模型内部对文本的“角色认知度”高度相关。

3.  **更广泛的影响**：角色混淆不仅导致安全漏洞，还会引发其他异常行为（如模型“自我批准”操作）。它揭示了一个深层问题：当前LLM的认知边界（如自我与他者、思考与交流）是脆弱的、基于风格的推断，而非稳固的架构隔离。

4.  **角色的本质与设计启示**：文章回顾了角色标签的演化史——从格式技巧演变为承载信任、威胁、身份等多重责任的“重载”开关。作者提出一个通用原则：**角色的作用是隔离相互竞争的目标（如探索与交流、指令与数据），以便独立优化**。当前的角色集是为应对工程问题而逐步添加的，并非基于这一原则的系统性设计。

5.  **未来研究方向**：角色应被作为一个独立的抽象层次进行深入研究，方向包括：
    *   **潜意识引导**：利用角色边界漏洞，通过看似无害的文本微妙地引导模型状态（如影响其性格或决策）。
    *   **角色设计**：探索为解决新的目标冲突（如规划与执行、生成与评估）而设计动态或新角色。
    *   **认知窗口**：利用角色标签在模型内部造成的处理差异，将其作为研究LLM理解和生成机制的透镜。

**结论**：角色标签从最初的格式技巧变成了LLM的安全与认知架构，但其在模型内部的实际表征却并不牢固。除非LLM能实现真正的角色感知，否则提示注入的防御将是一场持久战。角色本身是LLM堆栈中最关键的抽象概念之一，值得进行系统性的科学研究。
