# The Software Development Lifecycle Is Dead
- URL: https://boristane.com/blog/the-software-development-lifecycle-is-dead/
- Added At: 2026-06-13 14:21:49
- Tags: #read #agent

## TL;DR
AI代理正瓦解传统软件开发生命周期，将其转变为意图驱动的紧密循环。新技能是“上下文工程”，安全网依赖可观测性，行业需适应AI协作模式。

## Summary
文章指出，AI代理并非让软件开发生命周期（SDLC）提速，而是使其彻底瓦解。传统的SDLC各阶段（需求、设计、实现、测试、评审、部署、监控）原本是离散、顺序、依赖人工交接的，但在AI驱动的工作流中，这些阶段已合并为一个紧密的循环：意图 → 构建 → 观察 → 重复。

AI原生工程师不再遵循传统SDLC的仪式（如冲刺计划、故事点估算、PR评审），他们直接构建产品，跳过所有教条。需求不再是预先冻结的文档，而是通过迭代生成的副产品；系统设计从预先规划变为与AI实时协作发现；实现由AI代理完成；测试与代码同步生成，QA阶段消失；代码评审在AI生成大量PR的背景下成为瓶颈，未来将由自动化验证或对抗性代理完成；部署与发布解耦，实现持续部署；监控成为唯一存活的阶段，但必须进化为闭环系统，直接反馈给AI代理以自动修复问题。

新技能是“上下文工程”，即提供高质量上下文给AI代理，而非依赖流程或仪式。安全网是可观测性，但行业仍停留在传统仪表盘阶段，未适应AI驱动的快速迭代。作者认为，SDLC已死，未来属于意图驱动、紧密循环的AI协作模式。
