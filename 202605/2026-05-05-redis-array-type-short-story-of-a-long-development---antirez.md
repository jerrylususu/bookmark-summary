## Summary
本文讲述了 Redis 新数组数据类型的开发历程。作者从 1 月开始设计，历时四个月完成 PR 提交，期间大量使用 AI 辅助开发。

**设计与实现过程：**
- **第一月：** 与 AI 合作撰写详细规格文档，通过反复反馈优化设计。
- **第二月：** 初步实现后发现层级结构不足，决定采用更复杂的“超级目录”设计，以支持大范围索引的高效操作。
- **第三月：** 进行代码审查、优化和压力测试，确保实现的健壮性和性能。
- **后续扩展：** 因实际需求（如存储 Markdown 文件）引入了正则表达式功能（ARGREP），并优化了 TRE 库。

**核心体会：**
AI 提供了安全网，使作者能承担更复杂的任务，并确保代码质量，但高质量系统编程仍需开发者全程深度参与。新数组类型旨在让数值索引成为 Redis 数据语义的一部分，为更多用例打开可能。
