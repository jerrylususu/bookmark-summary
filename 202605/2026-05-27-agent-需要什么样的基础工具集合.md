# Agent 需要什么样的基础工具集合
- URL: https://jolestar.com/agent-basic-tool-set/
- Added At: 2026-05-27 14:46:01
- Tags: #read #agent

## TL;DR
文章探讨了 Agent 基础工具集的设计，指出 shell 能解决大部分问题，但编辑精确性和长任务调度仍是关键挑战，需平衡通用性与专用性。

## Summary
文章探讨了 Agent 基础工具集合的设计挑战，指出 shell 能解决 80% 的问题，但剩余 20% 的关键细节决定了 Agent 从 demo 走向实用系统。核心内容分为三部分：

1. **读操作：全走 shell 的取舍**  
   - Holon 废弃了专用读取工具（如 Read/Glob），全部通过 shell 完成，因为 shell 是 LLM 最熟悉的“编程语言”，降低模型认知负担。  
   - 代价是输出截断问题：框架限制 shell 返回长度，Agent 需多次读取大文件，而专用工具（如 Claude Code 的 Read）在边界场景更高效。

2. **写操作：编辑工具的演进与挑战**  
   - 简单编辑（如 sed）易出错，因此系统提供 Replace String 或 ApplyPatch 工具。Codex 的 ApplyPatch 配合 free grammar tool（直接传递原始文本，避免 JSON 转义）大幅降低错误率。  
   - Holon 兼容多模型：对支持 free grammar 的模型用原始 patch 格式，对其他模型用 git diff 格式，依赖模型对 diff 的训练数据积累。

3. **调度：长任务的生命周期管理**  
   - 长命令（如测试、构建）需自动转后台，避免阻塞或重复执行。业界共识是设置超时阈值，由 runtime 自动处理。  
   - 后台任务管理的三大难题：输出读取（轮询/推送）、任务停止（立即 kill 或优雅退出）、唤醒 Agent（需 runtime 与调度深度绑定），目前缺乏标准化方案。

总结：工具集设计需权衡通用性与专用性，shell 是基础但非万能。编辑精确性、patch 格式匹配和长任务调度是关键瓶颈，需进一步探索标准化方案。
