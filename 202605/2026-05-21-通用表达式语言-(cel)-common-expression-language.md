## TL;DR
通用表达式语言（CEL）是一种安全高效的表达式语言，支持一次编译、多次评估，适用于路由、安全策略等场景。其核心优势在于安全性与纳秒级评估速度，通过解析、检查和评估三阶段处理，广泛用于扩展声明性配置，如Google Cloud IAM的授权细化。

## Summary
通用表达式语言（CEL）是一种旨在实现快速、可移植且安全执行的表达式语言，适用于从路由远程过程调用到定义安全策略等多种场景。它支持一次编译、多次评估的工作流，特别适合性能要求严苛的应用。

CEL 的核心优势在于其安全性与速度。它允许安全执行用户代码，避免了类似 Python 中 `eval()` 的风险，同时通过阻止性能下降的行为，实现纳秒或微秒级的评估速度。

CEL 主要用于扩展声明性配置，例如在基于角色的访问控制（RBAC）中结合基于属性的访问控制（ABAC），以提供更强大的表达能力。Google Cloud IAM 就利用 CEL 表达式来细化授权范围。

CEL 的处理分为三个阶段：
1. **解析**：将人类可读的表达式转换为基于 Protocol Buffer 的抽象语法树（AST）。
2. **检查**：根据环境对表达式进行类型检查，确保变量和函数正确声明和使用。
3. **评估**：在运行时反复检索和评估存储的 AST，支持多次输入评估。

环境由服务定义，包括变量和函数的集合，确保表达式在安全且高效的上下文中执行。CEL 适用于各种应用，从简单布尔决策到构建复杂对象如 JSON 或协议缓冲区消息。
