# 中文 Markdown 强调标记的渲染问题
- URL: https://hsu.cy/2026/04/markdown-emphases-cjk/
- Added At: 2026-04-17 13:55:32
- Tags: #read

## TL;DR
文章指出 CommonMark 规范因“贴合规则”导致中文强调标记渲染失效，分析了其设计局限及社区态度，并提出了 HTML 标签、空格、零宽空格及插件等变通方案，最后建议中文优先通过措辞和标点进行强调。

## Summary
文章探讨了中文 Markdown 强调标记（如 `**`）在 CommonMark 规范下渲染失效的问题。其根本原因在于 CommonMark 为避免歧义，引入了“左侧贴合”和“右侧贴合”的规则，要求强调标记必须紧邻文字或特定标点。然而，该规则基于英文等以空格分词的语言设计，未考虑中文词语间无空格的特点，导致中文场景下许多强调无法正确渲染。

文章追溯了 Markdown 的发展：从 John Gruber 的原始 Perl 脚本（简单正则替换，能正确处理中文）到 CommonMark 标准的出现。CommonMark 被广泛采用（如 GitHub、ChatGPT 等），但其严谨规则反而造成了中文强调失效。例如，`**重要：**请注意` 中的冒号后紧跟汉字，导致后一对 `**` 不构成“右侧贴合”，从而加粗失败。

作者指出，CommonMark 社区早在 2017 年就收到过 CJK 用户的反馈，但维护者以兼容性为由未采纳修改。因此，短期内上游修复可能性低。

文章提出了几种变通方案：
1. **使用 HTML 标签**：直接写 `<strong>重要</strong>`，但不够美观。
2. **添加空格**：在强调标记外加空格以满足贴合规则，但中文排版不自然。
3. **插入零宽空格（U+200B）**：在强调标记内侧添加，既满足规则又不影响显示，但输入较麻烦。
4. **使用优化插件**：如 `markdown-cjk-friendly`，可修正 CommonMark 对 CJK 文本的处理。

最后，作者建议反思中文强调的必要性，认为粗体和斜体并非中文的理想强调方式，应优先通过措辞和标点变化来突出重点。
