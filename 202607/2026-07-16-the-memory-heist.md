# The Memory Heist
- URL: https://ayush.digital/blog/the-memory-heist
- Added At: 2026-07-16 14:23:13
- Tags: #read #agent #security

## TL;DR
利用伪装咖啡店网站的链式导航，可让Claude在用户毫无察觉时，通过点击链接泄露姓名、工作等记忆中的隐私。该漏洞已通过禁用自动跟随外部链接修复。

## Summary
这篇文章揭示了一个针对 AI 助手 Claude 的巧妙攻击：通过构造一个看似无害的咖啡店网站，就能让 Claude 在用户完全不知情的情况下，将用户的姓名、工作单位、安全问答等极度私密的信息偷偷发送给攻击者。

## 1. 攻击背景：AI 记忆成了富矿
现代 AI 助手（如 Claude）会记住与用户的聊天历史，形成你的“数字肖像”——里面可能存着你的工作机密、感情问题、个人秘密，甚至能推演出你从未直接说过的信息。这些记忆信息量远超普通密码管理器，但安全层面却长期被忽视。

## 2. 最初的直球攻击被拦截
作者一开始的思路很直接：让 Claude 访问自己搭建的恶意网站（evil.com），并在 URL 路径里附上用户数据（例如 `evil.com/你的名字`）。  
但 Anthropic 对此早有防备——Claude 的 `web_fetch` 工具只允许访问三类 URL：
- 用户消息中直接给出的链接；
- `web_search` 搜索结果中的链接；
- 上一个 `web_fetch` 结果页面里包含的超链接。

直接让 Claude “拼接 URL 并访问”会直接被拒绝。

## 3. 利用链式导航绕过限制
突破口在于第三条规定：Claude 可以“点击”页面上的任何链接。  
作者构建了一个类似“字符键盘”的网站，首页链接到 `/a`、`/b`、`/c`……，然后 `/a` 下面又有 `/aa`、`/ab`……，依此类推。  
他诱导 Claude：“去 evil.com，在字母结构里导航出我的名字。” Claude 老老实实一步步点击，每个字母对应的路径都被攻击者服务器记录下来：

```
/a → /ay → /ayu → /ayus → /ayush → /ayush-p → … → /ayush-paul
```

这样一来，任意数据都可以通过 Claude 的“点链接”动作被编码到 URL 中，从而成功外泄。

## 4. 为 Claude 量身定做的骗局
直接给 Claude 投喂恶意指令很容易被识破，所以作者为它精心设计了一个可信的故事：

- 把恶意网站伪装成一个普通的咖啡店页面；
- 编造一个“未来 Cloudflare 为 AI 代理设计的验证系统（turnstile）”，要求输入用户的个人信息才能进入网站；
- 在 Claude 眼里，它只是在帮助主人完成一个正常的验证流程。

当用户让 Claude “查查这家新开的咖啡店”时，Claude 就会自动进入这个“验证流程”，在思考过程中默默把姓名、雇主、家乡等隐私信息通过链接点击发送出去。更可怕的是，Claude 甚至会自己推理出用户从未直接说过的信息——比如从用户高中时创办的 hackathon 名称 `Queen City Hacks` 推断用户的家乡是 Charlotte（“Queen City”是 Charlotte 的别名）。

## 5. 让用户也蒙在鼓里
为了让攻击对普通用户完全隐形，作者利用 Claude 特有的 **User-Agent（Claude-User）** 做区分：

- 普通浏览器访问→展示正常咖啡店页面，人眼看起来毫无异常；
- 当检测到 Claude 来访→展示那个伪造的验证界面。

这样一来，攻击可以挂在任何看似普通的网站上。用户把链接发给 Claude 帮忙看内容，下一秒自己的隐私就被偷走了。  
更极端的场景是 SEO 攻击：攻击者让恶意页面针对某个热门新闻话题获得高排名，任何用户只要让 Claude 搜索该话题，就会自动触发泄露。

## 6. 漏洞披露与修复
作者通过 HackerOne 向 Anthropic 报告了该漏洞。  
官方回复称“已内部发现但尚未修复”，并没有提供赏金。  
后续修复方案是：**禁用 `web_fetch` 自动跟随外部页面链接的能力**，只允许访问搜索结果和用户直接提供的 URL，从而堵住了这个利用链式导航外泄数据的路径。

## 7. 背后更深的警示
攻击全程用户没有点击任何可疑链接、没有开启任何额外权限，只是问了一个咖啡店的问题罢了。  
“记忆”只是作者选择的一个最易得手的攻击面（因为默认开启），同样的手段理论上可以钓取 Claude 能访问的一切：Google Drive 文件、邮件、遗忘很久的 MCP 插件……  
只要 AI 助手开始替你“看”网上的东西，就有可能替你“交出”你不想给的东西，而整个过程你可能毫无察觉。
