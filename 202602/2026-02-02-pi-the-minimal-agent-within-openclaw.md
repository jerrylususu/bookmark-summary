## Summary
Pi 是由 Mario Zechner 开发的一个极简编码代理，作为 OpenClaw 项目的核心组件，强调通过代码编写和运行为核心的智能代理理念。文章由 Armin Ronacher 撰写，解释了他为何对 Pi 感到痴迷。

**概述**  
Pi 是一个轻量级编码代理，与 OpenClaw 项目紧密相关。OpenClaw 是一个流行的通信渠道代理，能够运行代码，而 Pi 是其底层实现的核心。Ronacher 使用 Pi 作为其主要工具，因为它简洁可靠。

**核心特点**  
- 极简设计：Pi 拥有最短的系统提示，仅提供四个基本工具（读取、写入、编辑和 Bash 命令）。  
- 扩展系统：通过扩展机制允许持久化状态，支持自定义功能开发。  
- 高可靠性：软件稳定，不闪退、内存占用低，由经验丰富的开发者编写。  
- 可定制性：用户可以基于 Pi 的组件构建自己的代理，如 OpenClaw 或 Telegram 机器人。

**设计哲学**  
Pi 遵循“代码编写代码”的理念，不支持 MCP（Model Context Protocol），而是鼓励用户让代理自行扩展功能，而非依赖外部下载。这体现了对 AI 能力的充分利用，强调代理的自适应性和扩展性。

**架构与功能**  
- 会话管理：Pi 使用树状会话结构，支持分支和热重载，便于测试和修复代码。  
- 状态持久化：扩展可以存储会话状态，支持消息跨模型提供商的可移植性。  
- 工具灵活性：虽然工具有限，但扩展可以添加自定义 TUI 组件，如进度条或交互式界面。

**扩展示例**  
Ronacher 开发了多个 Pi 扩展以提升体验：  
- `/answer`：提取和格式化代理问题，简化对话。  
- `/todos`：管理待办事项列表。  
- `/review`：用于代码审查，利用会话分支功能。  
- `/control`：实验性多代理系统。  
- `/files`：管理会话中涉及的文件。  
这些扩展大多由 Pi 代理根据用户要求自行构建，展示了代理的自我扩展能力。

**意义与未来**  
Pi 代表了软件构建软件的趋势，通过最小化核心和鼓励代理自我进化，预示了 AI 代理在未来开发中的重要性。OpenClaw 的成功证明了这种方法的潜力，Ronacher 认为这将成为未来的发展方向。
